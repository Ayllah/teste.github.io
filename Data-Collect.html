For this example, the Fantasia Database (fantasia) will be used. You can access it through the link: https://archive.physionet.org/cgi-bin/atm/ATM . Select options as shown below:

  • Database: Fantasia Database (fantasia)
  • Record: f2y10
  • Signals: all
  • Annotations: corrected beat annotations from an automated detector (ecg)
  • Length: to end
  • Time format: time/date
  • Data fromat: standard
  • Toolbox: Export signals as .mat

After following the instructions above, download the files that will appear: f2y10m.mat, f2y10m.info, f2y10m.hea and plotATM.m. 

To know the variables that are available in your file:

  • load f2y10m.mat
  • whos

Through the f2y10m.info file it is possible to obtain the following information:  fs = 250 Hz, Ts = 1/fs = 0.0004 s. â€‹This information is required by the CRSIDLab program, so a variable will be created.

  • fs = 250

Through this same file it is possible to notice that the respiration signal is on the first line, the ECG signal is on the second and the BP signal is on the third line. To collect them just follow the instructions below:

  • resp = val(1,:);
  • ecg = val(2,:);
  • bp = val(3,:);

To convert the RESP, BP and ECG data to physical units, you must create two variables "base = 0" and "gain = 409.6" and the data value must be subtracted from the base and divided by the gain from the following excerpt of code:

  • resp = (resp-base)./gain;
  • ecg = â€‹(ecg-base)./gain;
  • bp = â€‹(bp-base)./gain;

A time vector must also be created:

  • timeVec = (0:length(ecg)-1)/fs;

Saving the time, ecg and respiration vectors as well as the sampling frequency, in a .mat file:

  • save('sub_f2_y10.mat','fs','timeVec','ecg','resp', 'bp');

To open the file:

  • load('sub_f2_y10.mat');

After the file is opened, to plot the signals follow the instructions below:

  • figure; plot(timeVec,ecg);
  • figure; plot(timeVec,resp);
  • figure; plot(timeVec,bp);
